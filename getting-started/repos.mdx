GOV.UK maintains many repositories on GitHub under the [alphagov](https://github.com/alphagov) organisation. These include running applications, Ruby gems, utility scripts, infrastructure configuration and documentation.

## Repository types

Repositories are grouped into types based on their purpose:

<AccordionGroup>
  <Accordion title="Frontend apps">
    Applications that render pages to users on www.gov.uk. These receive requests from Router and read content from the Content Store API.

    Examples: `frontend`, `finder-frontend`, `collections`, `smart-answers`, `email-alert-frontend`, `feedback`
  </Accordion>
  <Accordion title="Publishing apps">
    Applications used by GOV.UK content editors to create and publish content. Secured behind Signon.

    Examples: `whitehall`, `publisher`, `specialist-publisher`, `travel-advice-publisher`, `collections-publisher`, `manuals-publisher`, `content-block-manager`
  </Accordion>
  <Accordion title="APIs">
    Applications that provide data and functionality to other applications in the stack.

    Examples: `publishing-api`, `content-store`, `search-api`, `search-api-v2`, `asset-manager`, `email-alert-api`, `account-api`, `locations-api`, `link-checker-api`
  </Accordion>
  <Accordion title="Supporting apps">
    Internal tools and infrastructure applications.

    Examples: `router`, `signon`, `release`, `authenticating-proxy`, `bouncer`, `search-admin`
  </Accordion>
  <Accordion title="Gems">
    Ruby gems used across GOV.UK applications. Published to RubyGems.

    Examples: `govuk_app_config`, `govuk_publishing_components`, `gds-sso`, `gds-api-adapters`, `govspeak`, `plek`, `rubocop-govuk`, `govuk_sidekiq`, `govuk_schemas`
  </Accordion>
  <Accordion title="Utilities">
    Scripts, tooling and infrastructure repositories that are not deployed as running applications.

    Examples: `govuk-docker`, `govuk-helm-charts`, `govuk-rfcs`, `govuk-user-reviewer`, `govuk-e2e-tests`, `govuk-fastly`, `govuk-mirror`, `govuk-infrastructure`, `govuk-ruby-images`
  </Accordion>
  <Accordion title="Data engineering">
    Repositories for data pipelines and analytics.

    Examples: `ga4-dataform`, `govuk-s3-mirror`, `govuk-knowledge-graph-gcp`, `govuk-knowledge-graph-search`
  </Accordion>
  <Accordion title="AI apps">
    Applications and experiments using AI and machine learning.

    Examples: `govuk-chat`, `govuk-ai-accelerator`
  </Accordion>
</AccordionGroup>

## Key repositories

| Repository | Type | Purpose |
|---|---|---|
| [publishing-api](https://github.com/alphagov/publishing-api) | API | Central API for publishing content to GOV.UK |
| [content-store](https://github.com/alphagov/content-store) | API | Stores and serves the latest edition of all content items |
| [router](https://github.com/alphagov/router) | Supporting app | Routes incoming requests to the correct frontend app |
| [whitehall](https://github.com/alphagov/whitehall) | Publishing app | Publishes ministerial and departmental content |
| [frontend](https://github.com/alphagov/frontend) | Frontend app | Renders many standard content formats including guides |
| [govuk-docker](https://github.com/alphagov/govuk-docker) | Utility | Local development environment using Docker Compose |
| [govuk-helm-charts](https://github.com/alphagov/govuk-helm-charts) | Utility | Helm charts for all GOV.UK Kubernetes deployments |
| [govuk_publishing_components](https://github.com/alphagov/govuk_publishing_components) | Gem | Shared UI component library for GOV.UK apps |
| [govuk_app_config](https://github.com/alphagov/govuk_app_config) | Gem | Shared configuration and utilities for GOV.UK Rails apps |
| [gds-api-adapters](https://github.com/alphagov/gds-api-adapters) | Gem | Ruby client libraries for GOV.UK APIs |
| [govspeak](https://github.com/alphagov/govspeak) | Gem | GOV.UK's Markdown dialect used in publishing apps |
| [govuk-rfcs](https://github.com/alphagov/govuk-rfcs) | Utility | Request for Comments — records of significant technical decisions |
| [signon](https://github.com/alphagov/signon) | Supporting app | Single sign-on and authorisation for publishing apps |
| [govuk-e2e-tests](https://github.com/alphagov/govuk-e2e-tests) | Utility | End-to-end test suite run in the integration environment |

## How repositories relate to applications

Not every repository is a deployed application. Some repositories — like gems and utilities — are used by many applications but are not themselves deployed.

The relationship between repositories and applications can be one-to-many: for example, [router](https://github.com/alphagov/router) is deployed as both `router` and `draft-router` in each environment.

## Finding your way around

- Browse repositories by type or team at [docs.publishing.service.gov.uk/repos.html](https://docs.publishing.service.gov.uk/repos.html)
- The [GOV.UK applications page](/getting-started/apps) lists deployed applications specifically
- The [Release app](https://release.publishing.service.gov.uk/) shows which version of each app is deployed to each environment

## Data download

Repository data is available as JSON or CSV:

- [repos.json](https://docs.publishing.service.gov.uk/repos.json)
- [repos.csv](https://docs.publishing.service.gov.uk/repos.csv)

In Google Sheets, import all repository data with:

```text
=importData("https://docs.publishing.service.gov.uk/repos.csv")
```

You can also view a [breakdown of repositories by team and type](https://docs.publishing.service.gov.uk/repos/by-team-and-type.html).
